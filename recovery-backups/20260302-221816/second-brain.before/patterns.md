## 네트워크 문제 진단 (2026-03-02)
- **[상황]** Telegram 사진 수신 실패(fetch failed 로그)
- **[판단]** 에이전트 잘못 아님 → 게이트웨이 네트워크/DNS 경로 이슈
- **[직관 신호]** 로그 패턴(fetch failed 반복) + 과거 네트워크 이슈 경험
- **[패턴명]** 로그 패턴 → 인프라 계층 이슈 우선 의심
- **[재사용 조건]** 반복되는 fetch/timeout 로그 + 에이전트 로직 정상 시 즉시 네트워크/DNS 계층 확인

## 크론 장애 분리 진단 (2026-03-01)
- **[상황]** NQ MACD cron 연속 실패(consecutiveErrors 누적)
- **[판단]** 단순 모델 실패가 아님 → 실행 주체 + 의존성 + 인증 체인 전체 검증 필요
- **[직관 신호]** group-sonnet에서 group-main 전환 후 정상 → 실행 주체 권한 이슈 + env-loader.sh 의존성 중복
- **[패턴명]** 크론 장애 분리 진단 패턴 (실행 주체 → 모델 → 의존성 → 인증 순서)
- **[재사용 조건]** 크론 연속 실패(consecutiveErrors >= 2) 시 실행 주체/모델/의존성 순서로 분리 검증

## 보안-가용성 트레이드오프 (2026-03-01)
- **[상황]** sandbox=all 적용 후 spawn docker ENOENT 장애
- **[판단]** 보안 강화는 런타임 전제(도커 설치)와 함께 검증해야 함
- **[직관 신호]** Docker 미설치 상태 → sandbox 실행 불가 → 보안 설정과 가용성 충돌
- **[패턴명]** 보안 설정 전제 검증 패턴
- **[재사용 조건]** 보안 강화(sandbox/firewall) 설정 적용 전 반드시 실행기 존재 여부(command -v docker 등) 사전 검증
