## 가능한 원인들

### 1. 환경 변수로 주입했었다가 세션이 바뀌며 사라짐
- 이전에 `OPENAI_API_KEY` 환경 변수로 키를 넣어서 썼을 수 있어.
- 세션이나 쉘이 바뀌면 환경 변수는 사라지니까, 다시 빈 상태로 돌아옴.

### 2. 다른 설정 파일이나 라우팅에 있었다
- OpenClaw가 provider를 여러 방식으로 읽을 수 있어.
- `openclaw.json`, `secrets/providers.json`, 환경 변수, `.env` 등.
- 예전엔 다른 쪽에 키가 있어서 작동했고, 지금은 그쪽이 비어 있을 수 있어.

### 3. 최근 업데이트나 설정 변경에서 덮어졌다
- OpenClaw나 다른 설정을 건드리면서 기존 키가 날아갔을 수 있어.
- 예: config patch, 재설치, 설정 초기화.

### 4. 실수로 지웠다
- 설정 파일을 편집하다 실수로 빈 문자열로 덮었을 수 있어.

## 가능성

1. **다른 설정 파일**에서 openai-codex를 정의해놓고 썼다가 그 파일이 비활성화됨
2. **임시로 환경 변수**에 토큰을 넣어서 썼다가 세션이 바뀌며 사라짐
3. **다른 시스템/계정**에서 GPT-5.4를 쓰고 있었고 이 시스템은 원래 안 됨

---

## 해결

GPT-5.4를 쓰려면 두 가지 중 하나를 해야 해:

### A. API 키 방식으로 전환
1. OpenAI API 키 발급
2. `secrets/providers.json`에 `openai_api_key` 추가
3. `models.providers.openai-codex` 섹션 정의
4. `openclaw gateway restart`

### B. OAuth 방식 복구
1. OpenAI OAuth 토큰 확보
2. `auth.profiles`에 openai-codex용 OAuth 프로필 추가
3. `models.providers.openai-codex` 섹션 정의
4. `openclaw gateway restart`

---

**어떤 방식으로 갈래?**  
- API 키가 더 간단해.  
- OAuth 토큰이 있다면 그걸 써도 되고.
