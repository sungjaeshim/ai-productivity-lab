## 설계 목표
좋은 입력이 들어왔을 때:

### Before
- 좋은 글 읽음
- inbox 쌓임
- 브리핑에서 한번 보임
- 잊힘

### After
- 좋은 글/인사이트 입력
- smart router가 분류
- action은 today/Todoist로
- learning은 learning card로
- 당일 회상 + 1/3/7/30 재호출
- 실제 적용 여부 체크
- 살아남은 것만 memory 승격 후보

## 1차에서 새로 필요한 중심축 2개

### A. Learning Card
학습 단위입니다.

- item_id
- title
- source
- summary
- 핵심 3개
- recall questions 3개
- connect 2개
- apply 1개
- status: new/reviewing/applied/transferred/stale

### B. Review Queue
타이밍 단위입니다.

필드 예시:
- card_id
- due_at
- review_type: d0/d1/d3/d7/d30
- mode: recall/explain/connect/apply
- status

즉,
- **카드 = 내용**
- **큐 = 다시 물을 시점**
