# 의사결정 패턴 모델
> 성재님의 과거 결정에서 추출한 패턴. 새로운 결정 시 참고.
> Updated: 2026-02-23

## 🧭 핵심 성향 (decisions.md 15건 분석)

### 1. 속도 > 완벽
- QMD v2: 로컬 GGUF 실패 → 즉시 OpenAI API 전환 (완벽한 로컬 고집 안 함)
- 모델 라우팅: 복잡하게 안 하고 채널별 3단 구분
- 글쓰기 템플릿: 2-3개만 먼저 → 판매 시작
- **패턴: MVP 먼저, 개선은 나중에**

### 2. 비용 민감하지만 시간 > 돈
- GLM 무료 우선, Opus는 핵심에만
- 하지만 Databento $49/월은 "데이터 품질이 시간 절약이면" 수용
- **패턴: 무료 가능하면 무료. 시간 절약이 명확하면 유료 OK**

### 3. 리스크 수용적 (but 되돌림 가능한 것만)
- NQ 선물 트레이딩 → 높은 리스크 수용
- 아파트 20개 전세 레버리지 → 큰 리스크 수용
- 하지만 퇴사는 GO 조건 충족 전까지 보류
- **패턴: 되돌릴 수 있으면 공격적, 되돌릴 수 없으면 신중**

### 4. 구조화 선호
- 항상 시스템/파이프라인으로 만들려 함
- 수동 반복 = 자동화 대상
- **패턴: 2번 이상 하면 자동화해야 한다**

### 5. 실용주의 (이론 < 결과)
- ML 백테스트: 과적합 방지 > 높은 수익률
- 블로그 SEO: 이론보다 실제 조회수
- **패턴: "되면 된 거다" 마인드**

## 📊 결정 속도 패턴
- 기술 선택 (모델, 도구): **즉결** (30분 이내)
- 비용 관련: **빠름** (당일, 시간>돈이면 바로)
- 인생 결정 (퇴사, 투자): **신중** (조건부 보류, 데이터 축적 후)
- 자녀 교육: **신중** (여러 관점 검토, 아내와 상의)

## 🔮 예측 규칙
새로운 결정 시 이 패턴으로 예측:

1. **되돌림 가능?** → Yes: 공격적 추천 / No: 보수적 추천
2. **시간 절약?** → Yes: 비용 들어도 추천 / No: 무료 대안 먼저
3. **2번 이상 반복?** → Yes: 자동화 추천
4. **MVP 가능?** → Yes: 작게 시작 추천 / No: 설계부터
5. **과거 유사 결정?** → decisions.md 검색 후 결과 참조

## 📝 업데이트 규칙
- 새 결정 추가 시 패턴 일치 여부 체크
- 패턴 위반하는 결정 → 새 패턴 후보로 기록
- 분기 1회 전체 리뷰 (주간 리포트에서)
