# 워크큐 관리 인사이트

## 발견: work-queue.md 활용 방법 정립 (2026-02-03)

### 문제점
- 작업이 "진행 중"과 "장기 프로젝트"로 분산
- 실제로 무엇을 해야 할지 불명확
- 장기 프로젝트가 너무 큼

### 해결책
**"명확한 다음 단계"가 워크큐의 핵심**

1. **장기 프로젝트는 "다음 작업" 하나만 노출**
2. **15-30분 단위로 쪼개기** (4시간마다 완료 가능)
3. **완료 후 자동으로 다음 단계 제안**
4. **실행 가능성**: "조사" → "구조 설계"처럼 구체적 행동

### 새로운 구조
```
🎯 오늘의 작업 (실행 가능한 단위)
- [ ] second-brain/ 인사이트 파일 1개 정리 (20분)
- [ ] 투자 전략 아이디어 1개 리서치 (30분)

🔄 진행 중 (큰 작업의 작은 단계)
- [ ] [자동매매] 백테스팅 코드 구조 설계 (1/3)

📋 장기 프로젝트 (다음 작업만 노출)
### 자동매매 전략 연구
- **다음 작업**: 백테스팅 코드 구조 설계
```

### 적용 상황
- 자율 작업 시간 (4시간마다)
- 큰 프로젝트 관리
- 자가 성장 루틴
