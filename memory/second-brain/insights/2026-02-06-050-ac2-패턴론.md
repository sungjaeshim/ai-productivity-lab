# 패턴론

> 성공 경험에서 반복되는 해결책을 추출하고 언어화

## 패턴이란?

### CA(크리스토퍼 알렉산더)의 정의
- 패턴은 발견하는 것
- 사람들이 직접 작성해야 함
- Best Practice 모음이 아님

### 핵심 원칙
> "같은 패턴을 만 번 적용하면, 만 개의 다른 결과가 나와야 한다"

- 같은 씨앗도 환경에 따라 다르게 자람
- 패턴 = 능동적 과정

---

## 패턴 포맷 (창준님 버전)

```
1. 제목 (솔루션 표현)
   - 이름이 중요! 대화에서 쓸 수 있어야
   
2. 상징/그림

3. 문제상황
   - 언제 이 패턴이 필요한가?

4. 맥락
   - 적용 가능한 상황

5. 해결책 (시퀀스)
   - 단계별 행동

6. 예시 (스토리텔링)
   - 실제 적용 사례

7. 주의점/부작용

8. 관련 패턴
```

---

## 패턴 활용처

1. **조직 지식 관리**: 암묵지 명시화
2. **나의 암묵지 분석**: CTA(Cognitive Task Analysis)
3. **커뮤니티 개발**: 공유된 언어 만들기
4. **신규입사자 적응**: 핵심 패턴 전수
5. **개인 성장**: 성공 경험에서 패턴 추출

---

## 패턴 추출 방법

### 1. 성공 경험 수집
- 여러 번 성공한 경험
- 한 번만 해봐도 패턴 있을 수 있음

### 2. 공통점 찾기
- 상황은 달랐는데 해결책은 비슷했던 것

### 3. 언어화
- 제목 붙이기 (가장 중요!)
- 문제상황 정의
- 해결책 시퀀스화

### 4. 검증
- 다른 상황에 적용해보기
- 피드백 받기

---

## 패턴 실습 아이디어 4가지

1. **아이에게 가르칠 때 내 패턴 뽑기**
2. **지인의 일 이해하기**: 1시간 인터뷰로 패턴 3-5개
3. **잘하고 싶은 일**: 잘하는 사람 인터뷰
4. **초보자를 위한 대표패턴 만들기**

---

## 패턴 언어

> "단어들을 모아 문장을 만들듯, 패턴도 엮어서 문맥(패턴 언어)을 만든다"

### 핵심
- 패턴 하나에만 집중하는 것이 문제
- 패턴들이 엮여야 좋은 결과
- 시퀀스(순서)가 중요

---

## 안티패턴 주의

- 안티패턴은 다루지 않기를 권함
- 패턴은 "성공적인 것"을 찾는 것
- Problem과 Solution의 scope 수렴이 어려움
