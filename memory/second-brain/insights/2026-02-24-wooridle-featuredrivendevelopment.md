- **핵심**: FDD에 대해 알아보면서, 모델링을 미리 한다는 점, 기능 목록 위주의 계획을 한다는 점 등이 실제 프로젝트에서 진행하는 방식과 일치하는 부분이 좀 있는 것 같다는 생각이 들었고, 그래서 실제 프로젝트에 좀 더 적합한 방식이라는 생각이 들었다
- **주요 내용**:
  - FDD Process Develop Model Build Feature List Planning Design by Feature Build by Feature 1~3 단계는 시동 단계라고 할 수 있고, 4~5 단계는 구현 단계라고 할 수 있다...
  - 4- 5- 4- 5 이렇게 iterative하게 진행한다...
  - Develop Model 이 단계에서는 세 가지의 역할이 필요하다...
