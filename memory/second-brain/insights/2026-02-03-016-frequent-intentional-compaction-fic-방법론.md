# Frequent Intentional Compaction (FIC) 방법론

*Human Layer 팀의 AI 코딩 워크플로우에서 추출*

---

## 핵심 원리

**"컨텍스트 활용률을 40% 이하로 유지하며, 의도적으로 컨텍스트를 압축하는 것"**

---

## 3단계 워크플로우

### 1. Research (조사)
- **목표**: 코드베이스와 문제 이해 (코드 작성 X)
- **산출물**: 연구 파일 (`RESEARCH.md`)
  - 관련 파일 목록
  - 함수 매핑
  - 데이터 플로우 다이어그램
  - 문제 영역의 요약
- **효과**: AI가 매번 거대 코드베이스를 다시 읽지 않아도 됨

### 2. Plan (계획)
- **목표**: 명시적인 구현 계획 수립
- **산출물**: 계획 파일 (`PLAN.md`)
  - 수정할 파일 목록
  - 구현 순서
  - 검증 방법
- **중요**: 나쁜 계획 = 수천 줄의 나쁜 코드

### 3. Implement (구현)
- **목표**: 계획 실행 + **Frequent Intentional Compaction**
- **기법**:
  - 컨텍스트 활용률 40% 미만 유지
  - 완료된 작업은 "완료" 표시 후 컨텍스트에서 제거
  - 남은 작업만 컨텍스트에 유지
  - 진행 상황에 따라 계획 업데이트

---

## 인간의 새로운 역할

### 코드 대신 계획 검토
- ❌ 수천 줄의 AI 생성 코드 리뷰
- ✅ 연구 파일과 계획 파일 리뷰
- 훨씬 간결하고 이해하기 쉬움
- 초기에 문제 발견 가능

### 추상화 수준에 집중
- 가장 큰 인사이트 = 가장 높은 추상화 수준
- 초기 연구의 오해해석 → 수천 줄의 나쁜 코드
- 구현 계획의 작은 오류 → 수백 줄의 영향
- **시간 투자 비중**: Research > Plan > Implement

---

## 실전 성과

- 300,000줄 코드베이스에서 중요 변경
- 7시간 세션에서 35,000줄 코드 배포
- 40-60% 컨텍스트 활용률 최적화

---

## 적용 원칙 (자비스용)

1. **큰 작업 시작 전**: RESEARCH.md 생성
2. **코드 작성 전**: PLAN.md 생성
3. **진행 중**: 컨텍스트 40% 미만 유지
4. **완료된 것**: 즉시 컨텍스트에서 제거
