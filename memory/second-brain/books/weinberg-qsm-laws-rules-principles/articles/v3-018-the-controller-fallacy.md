# V3-018 — The Controller Fallacy

## Meta
- Volume: 3 (Congruent Action)
- Original: **The Controller Fallacy**
- Statement: "If the controller isn’t busy, it’s not doing a good job; if very busy, it must be good." (p.128)
- KR 제목: **컨트롤러 오류: 바쁨을 성과로 착각하는 오류**
- Confidence: High

## 1) 정밀 번역
컨트롤러가 바쁘지 않으면 일을 못하는 것이고, 매우 바쁘면 좋은 컨트롤러라는 생각은 오류다.

## 2) 핵심 정의 3줄
1. 바쁨은 성과가 아니라 상태 신호일 뿐이다.
2. 좋은 제어는 오히려 사건 수를 줄여 ‘한가해 보일’ 수 있다.
3. 바쁨 중심 평가는 소방 영웅주의를 강화한다.

## 3) 왜 중요한가
- 운영/관리 성과가 ‘분주함’ 기준으로 왜곡되는 것을 막는다.

## 4) 실무 적용 체크리스트
- [ ] 성과평가에서 바쁨 지표 대신 안정성 결과 지표를 쓰는가?
- [ ] 예방 활동을 공식 KPI로 반영하는가?

## 5) 오해/한계
- 오해: "한가하면 무능."
- 한계: 실제 무능 가능성도 있어 결과 데이터와 함께 판단해야 한다.
