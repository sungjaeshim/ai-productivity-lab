# V1-030 — The Controller Fallacy

## Meta
- Volume: 1 (Systems Thinking)
- Original: **The Controller Fallacy**
- Statement: "If the controller isn’t busy, it’s not doing a good job. If very busy, it must be good." (p.197)
- KR 제목: **컨트롤러 오류: 바쁨을 유능함으로 착각하는 판단 오류**
- Confidence: High
- Tags: #fallacy #management #busyness #control

## 1) 정밀 번역
컨트롤러가 바쁘지 않으면 일을 못하는 것이고, 매우 바쁘면 좋은 컨트롤러라는 생각은 오류다.

## 2) 핵심 정의 3줄
1. 바쁨은 성과가 아니라 상태일 뿐이다.
2. 좋은 통제는 시스템 안정성을 높이고 소방 활동을 줄인다.
3. 바쁨 기반 평가는 나쁜 행동(과잉개입, 소방영웅주의)을 유도한다.

## 3) 왜 중요한가
- 운영 조직이 예방보다 사건 중심 행동으로 기울어지는 원인을 설명한다.

## 4) 실무 적용 체크리스트
- [ ] 개인 성과를 바쁨이 아니라 결과 지표로 평가하는가?
- [ ] 긴급업무 비중을 낮추는 구조적 개선을 보상하는가?

## 5) 오해/한계
- 오해: “바쁘면 유능.”
- 한계: 바쁨 데이터는 용량계획엔 유용하나 성과판정엔 부적절.
