# V2-009 — Counting Lines of Code As a Metric

## Meta
- Volume: 2 (First-Order Measurement)

- Statement: "Any technique that visualizes something meaningless leads to distortion." (p.58)
- KR 제목: **LOC 지표 함정: 무의미한 것을 계량하면 왜곡이 발생한다**
- Confidence: High
- Tags: #loc #metrics #distortion

## 1) 정밀 번역
무의미한 대상을 가시화(계량)하는 어떤 기법도 왜곡으로 이어진다.

## 2) 핵심 정의 3줄
1. LOC 같은 대리 지표는 생산성/가치와 쉽게 분리된다.
2. 계량 가능성은 중요성의 증거가 아니다.
3. 의미 없는 측정은 행동을 나쁜 방향으로 유도한다.

## 3) 왜 중요한가
- 코드 줄 수 최적화가 오히려 복잡도/부채를 늘릴 수 있다.

## 4) 실무 적용 체크리스트
- [ ] 지표가 실제 목표(가치/신뢰성)와 인과적으로 연결되는가?
- [ ] 대리 지표를 단독 평가에 사용하지 않는가?

## 5) 오해/한계
- 오해: "숫자화되면 관리가 쉬워진다."
- 한계: 실무에서 대리 지표는 필요하나 보조적 용도로 제한해야 한다.
