# V1-035 — The Army Principle (Modified)

## Meta
- Volume: 1 (Systems Thinking)
- Original: **The Army Principle (Modified)**
- Statement: "There are no bad programmers; there are only bad managers who don’t understand failure dynamics." (p.213)
- KR 제목: **군대 원리(수정): 실패 동학을 모르는 관리가 문제를 만든다**
- Confidence: High
- Tags: #software-management #failure-dynamics #leadership

## 1) 정밀 번역
나쁜 프로그래머는 없고, 실패의 동학을 이해하지 못하는 나쁜 관리자만 있다.

## 2) 핵심 정의 3줄
1. 개발 실패는 개인 코딩 실력보다 관리 구조 문제인 경우가 많다.
2. 실패 동학(압박-왜곡-통제상실)을 이해해야 반복 실패를 줄인다.
3. 관리 역량은 일정/리소스 배분 이상의 시스템 설계 역량이다.

## 3) 왜 중요한가
- 같은 실수 반복은 개인 교정이 아니라 운영 구조 교정 신호다.

## 4) 실무 적용 체크리스트
- [ ] 관리자가 실패 패턴/선행신호를 주기적으로 리뷰하는가?
- [ ] 일정 압박 시 범위 조정/품질 보호 규칙이 있는가?

## 5) 오해/한계
- 오해: “개발자 역량은 무관.”
- 한계: 개인 역량과 관리 역량은 둘 다 필요하나 재발 방지의 레버는 관리 측면이 크다.
