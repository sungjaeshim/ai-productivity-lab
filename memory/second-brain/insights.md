- **배운 점**:

  - 브라우저는 항상 자동 실행이 아니라 **도구 호출 시 조건부 실행**

- **액션**:
  - OpenManus 실전운영 인사이트 문서화 및 재사용 프롬프트 템플릿 저장
  - 3단계 도입전략(PoC→운영→프로덕션) 기준으로 안내
- **상세 문서**: `memory/second-brain/insights/2026-03-04-OpenManus-실전운영-인사이트.md`

## 2026-02-22

### AI 시대 성장 아이러니: 기회 평등 ≠ 결과 평등
- **상황**: 바둑 AI 도입 후 상위-하위 격차 확대 (EBS 다큐)
- **배운 점**:
  - 의도적 연습 = 어려운 과제 + 실행 + 피드백 + 수정 + 재실행 (단순 반복 X)
  - AI 아이러니: 답은 평등한데, 해석/흡수 능력은 실력 따라 다름
  - 실력 높을수록 AI 해법을 더 정확히 읽고 자신의 것으로 만듦
  - 성패 = AI 쓰느냐 X, AI를 "피드백 장치"로 설계해 의도적 연습 루프 안에서 굴리느냐
- **액션**:
  - AI를 답 제공자가 아니라 피드백 장치로 활용
  - 도전 → 피드백 → 재실행 루프 구조화
  - AI 답을 받고 "왜?" 질문하기
- **태그**: #AI #성장 #학습 #의도적연습 #피드백

### AI 에이전트 표준 전쟁의 핵심 구조
- **상황**: NIST AI 에이전트 표준화 이니셔티브 발표 (2026-02-19)
- **배운 점**:
  - 표준 = 기술 사양서 X, "누가 이 세계의 시민인지 결정하는 경계선"
  - TCP/IP의 "누가 보냈는지" 생략 → 30년 보안 부채. AI 표준도 같은 실수 가능
  - 에이전트 아이덴티티 + 온체인 = 결제 레일 장악. 네트워크 이펙트 승자독식
  - OSI vs TCP/IP: 완벽한 설계 < 돌아가는 코드
  - "자발적 표준이 강제가 되는 임계점" 존재. 영어, 미터법, TCP/IP처럼
- **액션**:
  - MCP(Anthropic), ERC-8004, .agent TLD 후보군 모니터링
  - 한국의 표준 논의 참여/발언권 확인
  - 에이전트 경제에서 네트워크 이펙트 기반이 될 레이어 식별
- **태그**: #AI #표준화 #전략 #네트워크이펙트 #기술패권

### 시스템 자동화의 3단계 성숙도
1. **Level 1**: 실행 자동화 (크론으로 돌림) ← 현재 대부분 여기
2. **Level 2**: 피드백 자동화 (결과 수집 + 자동 분석) ← 일부만 됨 (Ralph Loop)
3. **Level 3**: 자가 개선 자동화 (분석 → 자동 수정) ← 아직 없음
- 현재 14개 크론 중 Level 2 도달: Ralph Loop, 시스템 모니터링 (2개)

#### 1. ORID 퍼실리테이션 기법
**용도**: 그룹 미팅, 코칭, 고민 구체화

질문 순서:
- **O** (Objective)  : "요즘 실제로 어떤 일이 일어나고 있어요?" ← 사실 수집
- **R** (Reflective) : "그게 어떤 느낌이에요? 힘든 부분은?" ← 감정/반응
- **I** (Interpret)  : "왜 그게 문제라고 생각하세요?" ← 의미 해석
- **D** (Decisional) : "그럼 가장 먼저 뭘 바꾸고 싶어요?" ← 결정/액션

**핵심**: 답을 주기 전에 ORID로 물으면 상대가 스스로 답을 찾는다. AI 답변보다 먼저 써야 할 기법.
**적용**: 스타트업 대표 고민 청취, 1:1 코칭, 팀 회의 진행

---

#### 2. ADKAR 변화관리 모델
**용도**: AI 도입, 조직 변화, 습관 형성

| 단계 | 의미 | AI 도입 맥락 |
|------|------|-------------|
| **A**wareness | 변화 필요성 인식 | "AI 안 쓰면 뒤처진다" |
| **D**esire | 변화하고 싶은 욕구 | "나도 써보고 싶다 (FOMO)" |
| **K**nowledge | 어떻게 바꿀지 앎 | "어떻게 쓰는지 모른다" |
| **A**bility | 실제로 할 수 있음 | "직접 해볼 자신이 없다" |
| **R**einforcement | 지속하게 하는 환경 | "계속 쓰게 만들기" |

**핵심**: AI 도입 실패는 기술 문제가 아니라 ADKAR 어느 단계에서 막혔는지의 문제.
**적용**: AI 컨설팅 진단 프레임, 조직 변화 코칭

---

#### 3. MINDSPACE 넛지 프레임워크 (영국 정부 + 심리학자 개발)
**용도**: 행동 변화 설계, 습관 형성, 선택 설계

| 요소 | 의미 | 적용 예시 |
|------|------|----------|
| **M**essenger | 누가 전달하느냐가 중요 | 신뢰받는 사람이 AI 추천해야 도입됨 |
| **I**ncentives | 손실 회피 심리 | "안 쓰면 경쟁사가 앞선다" 프레이밍 |
| **N**orms | 남들이 하는 걸 따라감 | "다른 스타트업들은 이미 쓰고 있어요" |
| **D**efaults | 기본 설정을 따라감 | AI 툴을 기본 워크플로에 넣기 |
| **S**alience | 나와 연관된 것에 주목 | 내 업종 사례만 골라서 보여주기 |
| **P**riming | 무의식 단서가 행동 유발 | "빠른 AI" vs "스마트 AI" 언어 선택 |
| **A**ffect | 정서적 연대가 행동 형성 | AI 도입 성공 스토리 감정 이입 |
| **C**ommitment | 공개 선언하면 지킴 | "다음 주까지 ChatGPT 써보겠습니다" |
| **E**go | 더 나은 존재로 여겨지길 원함 | "AI 쓰는 리더" 정체성 부여 |

**핵심**: 아는 것과 행동하는 것 사이의 간극 = MINDSPACE로 설명됨. Nudge는 강요 없이 행동을 유도.
**출처**: 김창준 AC2 메일링리스트 (2013) + 영국 정부 공개 PDF

---

#### 연결 인사이트
- ORID → ADKAR → MINDSPACE 순서로 쓰면 최강 변화 유도 시퀀스
  1. ORID: 문제를 스스로 정의하게
  2. ADKAR: 어느 단계에서 막혔는지 진단
  3. MINDSPACE: 막힌 단계를 행동으로 뚫는 설계
- **오늘 저녁 6인 모임**: ORID로 고민 수집 → ADKAR으로 진단 → 성재가 K(지식) 단계 해결
- AC2의 핵심은 "행동 변화 = 知 + 設計 (지식 + 선택 설계)". MINDSPACE가 설계 파트.

#### 4. Effectuation (창업가적 사고법) ⭐⭐⭐ 오늘 저녁 핵심
**출처**: Saras Sarasvathy, 버지니아大 연구 (45명 성공 창업가 인터뷰)

**핵심 대비**:
| Causation (MBA식) | Effectuation (창업가식) |
|-----------------|---------------------|
| 목표 먼저 → 최선 방법 탐색 | 지금 가진 것 먼저 → 가능성 탐색 |
| 미래 예측 → 통제 | 통제 → 예측 불필요 |
| 경쟁사 분석 | 파트너로 인식 |
| 얼마나 얻을 수 있나? | 잃어도 괜찮은 최대치는? |

**5 Principles**:
- **Birds-in-Hand**: 지금 내가 가진 것(기술/팀/네트워크)으로 뭘 할 수 있나?
- **Affordable Loss**: 얼마나 잃어도 괜찮은가? → AI MVP의 철학적 근거
- **Lemonade**: 위기를 기회로 (레몬이 생기면 레모네이드를)
- **Patchwork Quilt**: 경쟁사 → 파트너로 프레이밍
- **Pilot-in-the-plane**: 트렌드 예측 말고 내가 통제할 수 있는 것에 집중

**오늘 저녁 활용**:
> "AI 도입, 목표부터 잡지 마세요. 지금 팀에 있는 것 중 AI 붙이면 가장 빠른 게 뭐예요?" (Birds-in-Hand)
> "AI가 실패해도 괜찮은 가장 작은 실험이 뭐예요?" (Affordable Loss = AI MVP)

#### 5. Double Loop Learning (이중 루프 학습)
**출처**: Chris Argyris (하버드)

| Single Loop | Double Loop |
|-------------|-------------|
| 문제 발생 → 방법 수정 | 문제 발생 → **전제/가정 자체를 의심** |
| "어떻게 더 잘할까?" | "왜 이걸 하고 있나?" |
| 오류 수정 | 패러다임 전환 |

**AI 도입 적용**:
- Single Loop: "ChatGPT 어떻게 더 잘 쓸까?"
- Double Loop: "우리 비즈니스 모델에서 AI가 정말 필요한 부분이 어딘가? 전제 자체를 바꿔야 하나?"
- AI 도입 실패 대부분 = Single Loop에서만 해결하려다 실패

#### 6. Project Oxygen (구글의 위대한 관리자 연구)
**출처**: Google rework 연구

- 2002년 구글이 관리자 전원 제거 실험 → 실패
- 2008년 "관리자 불필요" 연구 → 오히려 반대 결과 발견
- 탁월한 관리자 공통점: **코칭, 의사결정, 협업, 팀 임파워링, 비전 공유**
- 핵심: 관리자 역할 = 팀이 스스로 할 수 있도록 돕는 것 (퍼실리테이터와 동일)

**AI 시대 적용**: AI = 팀의 관리자 역할 일부를 보조. 단 "코칭, 비전 공유"는 사람만 가능.

#### 7. 학습조직 (Learning Organization)
**출처**: Peter Senge, The Fifth Discipline (1990)

5 Disciplines:
1. **Personal Mastery**: 개인 성장 의지
2. **Mental Models**: 자신의 가정과 편견 인식
3. **Shared Vision**: 공동 목표
4. **Team Learning**: 팀 단위 학습
5. **Systems Thinking**: 시스템 전체를 보는 눈 ← 핵심

**AI 시대 적용**: AI 도입 성공 스타트업 = 학습조직. 개인이 AI 잘 쓰는 게 아니라 **팀 전체가 AI로 학습하는 구조** 만드는 것.
