# JARVIS_VISUALIZATION_WORKFLOW_SOP

작성일: 2026-03-11
상태: active

## 목적

시각화/다이어그램/개념지도 요청을 받을 때,
단순 코드 생성으로 끝내지 않고
**초안 → 렌더 → 텔레그램 검수 → 최적화 → 노션 정리**까지
연결된 흐름으로 완성하는 운영 원칙을 고정한다.

---

## 사용자 선호 요약

사용자는 다음 흐름을 매우 선호한다:

1. 초안 생성
2. Mermaid CLI 등으로 실제 렌더
3. 텔레그램에서 바로 확인/검수
4. 가독성 중심 수정
5. 최종본을 노션에 정리

핵심은 **"코드 제공"보다 "바로 보고, 수정하고, 보관되는 결과"** 이다.

---

## 기본 운영 원칙

### 1. 결과물은 사용 가능 상태까지 간다
- Mermaid 코드만 주고 끝내지 않는다.
- 가능하면 PNG/PDF까지 렌더한다.
- 사용자가 실제 보는 환경(특히 Telegram 모바일)에서 검수 가능하게 한다.
- 최종본은 Notion에 정리해 재사용 가능한 자산으로 남긴다.

### 2. 초안은 빨리, 최적화는 반복적으로
- 첫 버전은 완벽함보다 **빠른 시각화**가 우선이다.
- 실제 렌더 후 사용자 피드백을 반영해 구조를 줄이고 다듬는다.

### 3. 모바일 가독성을 우선한다
특별한 요청이 없으면:
- 세로형 우선
- 연결선 최소화
- 중심 노드 강조
- 파생 항목은 묶어서 단순화
- 한 화면에서 핵심 구조가 보이도록 설계

### 4. Notion은 최종 배포/아카이브 레이어다
Notion 페이지에는 가능하면 아래를 포함한다:
- 제목
- 목적
- 사용법
- 질문 예시
- Mermaid 원본
- 산출물 정보(PNG/PDF 등)

## 표준 작업 흐름

## Phase 1. 구조화
요청을 받으면 먼저 정리한다:
- 목적이 무엇인가
- 최종 사용처가 어디인가 (Telegram / Notion / 발표 / 문서)
- 모바일 우선인가
- 학습용인가 발표용인가
- 전체지도인가 축약판인가

## Phase 2. 초안 생성
- 핵심 개념과 하위 개념 정리
- 연결선 우선순위 지정
- Mermaid 초안 작성
- 너무 복잡하게 시작하지 않기

## Phase 3. 렌더 및 검수
- Mermaid CLI로 PNG 렌더
- 필요 시 PDF도 생성
- Telegram에 바로 보내 실제 가독성 확인

## Phase 4. 가독성 최적화
주요 조정 항목:
- 노드 수 줄이기
- 파생 개념 묶기
- 연결선 수 줄이기
- 세로형 재배치
- 제목/읽는 법 추가
- 모바일 기준 폰트/간격 개선

## Phase 5. 최종본 패키징
최종적으로 정리할 수 있는 항목:
- Mermaid 원본
- PNG
- PDF
- 짧은 설명 문구
- AI 질문 예시
- Notion 페이지

---

## 텔레그램 검수 운영 원칙

Telegram은 1차 QA 화면으로 본다.

검수 질문 예시:
- 잘 보이는가?
- 너무 복잡한가?
- 어떤 버전이 더 낫나?
- 더 크게/더 단순하게 갈 필요가 있나?

검수 후에는 설명보다 구조 조정을 우선한다.

---

## Notion 운영 원칙

테스트용이 아니라 최종본을 만들 때는 가능하면 아래 구조를 사용한다.

1. 제목
2. 목적
3. 읽는 법
4. 핵심축 요약
5. AI에게 던질 질문 예시
6. Mermaid 원본
7. 산출물 정보

Notion 기능이 허용되면 Mermaid 뷰어/블록으로 전환해 사용할 수 있다.

---

## 품질 기준

좋은 결과물의 기준은 다음과 같다:
- 코드가 아니라 **화면에서 읽힌다**
- 사용자가 **다음 행동을 바로 떠올릴 수 있다**
- 최종본이 **다시 꺼내 써도 맥락이 살아 있다**
- 결과가 **텔레그램 검수 + 노션 보관**까지 연결된다

## 앞으로의 기본 제안 문구

비슷한 요청이 오면 아래 흐름으로 먼저 제안한다:

> 먼저 초안 하나 만들고,
> 바로 렌더해서 텔레그램에서 보시게 한 뒤,
> 모바일 가독성 피드백 반영하고,
> 최종본은 노션에 정리해둘게요.

## 향후 발전 방향

### A. 템플릿화
반복 구조를 템플릿으로 고정:
- 제목
- 목적
- Mermaid 원본
- PNG/PDF
- 질문 예시
- Notion 페이지 구조

### B. 스크립트화
가능하면 자동화:
- .mmd 입력 → PNG/PDF 출력
- Telegram 검수본 생성
- Notion 페이지 생성 보조

### C. 유형별 프리셋
향후 아래 유형별 기본 구조를 만들 수 있다:
- 학습용 개념 지도
- 발표용 다이어그램
- 운영 흐름도
- 의사결정 지도
