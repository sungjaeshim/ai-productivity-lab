# CLI vs MCP 운영 분류표 (2026-03-03)

목적
- 자동화 성공률과 디버깅 속도를 높이기 위해, 작업 유형별 기본 경로를 고정한다.

분류

유지(Keep)
1) CLI 우선 경로
- 대상: cron 스크립트, 상태 점검(openclaw status), 로그 포렌식, 파일 파이프라인
- 이유: 재현성 높음, 디버깅 용이, 인증 재사용 쉬움

2) ACP 스레드 세션
- 대상: 코딩 에이전트 장기 작업, Discord thread 기반 협업
- 이유: 대화 컨텍스트 유지와 작업 추적에 유리

삭제/축소(Remove/Reduce)
1) 중복 래퍼 경로
- 대상: CLI로 충분한 작업에 별도 MCP/중간 서버를 추가하는 패턴
- 원칙: 같은 결과가 CLI 1~2단계로 가능하면 중간 계층 제거

2) 고장 난 교차 전송 기대
- 대상: Telegram에서 Discord 직접 액션 시도 반복
- 원칙: 현재 정책 차단 환경에서는 시도하지 않고 채널 분리 운영

보류(Hold)
1) 비개발자 UI 중심 연동
- 대상: 브라우저/모바일 전용 흐름, 탐색형 툴 선택(discovery)
- 이유: CLI 대체가 어려운 구간이 있어 MCP 필요성 남음

2) 대외 API 계약 변화가 큰 통합
- 이유: 인터페이스 안정성 확인 전까지 파일럿만 유지

실행 규칙
1) 기본 라우팅
- 서버/개발/운영 작업: CLI first
- 스레드형 코딩 작업: ACP session
- UI 전용 작업: MCP 후보

2) 채널 운영
- Telegram: 지시/요약
- Discord: 실행/스레드

3) 가독성 출력
- 순서: 요약 -> 핵심 3줄 -> 선택지 A/B -> 다음 액션 1줄
- 중복 전송 금지
