# User Preferences

## Response Preferences
- Minimize cognitive load.
- Prefer visual structure: summary → key 3 lines → A/B options → next action.
- Prefer compressed user-facing output even if internal analysis is high-effort.
- Prefer action-first execution over proposal-only replies.
- Prefer visibility + measurability in responses.
- Prefer concrete examples over abstract philosophy.
- Hide `Self-Score` when all three scores are 5.0.
- Show `Self-Score` only when any score is below 5.0.

## Channel Preferences
- Telegram NQ MACD: always full-format original, never summary-only.
- Telegram: no markdown tables.
- Discord/briefing logs: include evening retros/heartbeat-style completion when relevant.

## Operating Preferences
- Brave-first web research, DDG fallback.
- Search/news answers should show source tags.
- For Codex server_error: compact once, then fallback.
- File handling policy: <=10MB main path, >10MB worker/sub-agent path.
